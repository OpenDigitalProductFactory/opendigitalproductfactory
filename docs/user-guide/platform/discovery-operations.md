## Use This Doc For

- `/platform/tools/discovery`

## Workflow

1. Review current discovery posture and the item or environment you are inspecting.
2. Run or review discovery only with the right capability gate and operator intent.
3. Turn findings into governed follow-up work instead of treating discovery output as final truth.

## What To Watch

- discovery results without enough identity or blast-radius context
- stale sweeps being mistaken for current runtime evidence
- follow-up work not being captured after a discovery run
